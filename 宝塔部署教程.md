# 宝塔面板部署教程

本教程指导如何在宝塔面板上部署「二次元地址发布系统」。

---

## 环境要求

| 组件 | 版本要求 | 推荐配置 |
|------|---------|---------|
| PHP | 7.4+ | 8.0+ |
| MySQL | 5.6+ | 8.0 |
| Nginx/Apache | 任意 | Nginx 1.20+ |
| 系统 | Linux | CentOS 7+/Ubuntu 20.04+ |

---

## 步骤一：安装宝塔面板

### 1.1 安装命令

**CentOS/RedHat:**
```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484b
```

**Ubuntu/Debian:**
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484b
```

### 1.2 记录登录信息

安装完成后会显示：
- 外网面板地址
- 内网面板地址
- 用户名
- 密码

**请务必保存好这些信息！**

---

## 步骤二：安装 LNMP 环境

### 2.1 登录宝塔面板

访问面板地址，使用账号密码登录。

### 2.2 安装套件

进入 **软件商店**，安装以下软件：

| 软件 | 版本 | 安装方式 |
|------|------|---------|
| Nginx | 1.20+ | 极速安装/编译安装 |
| MySQL | 5.7 或 8.0 | 极速安装 |
| PHP | 8.0 或 8.1 | 极速安装 |
| phpMyAdmin | 任意 | 极速安装 |

> **注意**: PHP 版本建议选择 8.0 或 8.1，兼容性最好。

### 2.3 等待安装完成

安装时间约 5-10 分钟，安装完成后在 **首页** 查看服务状态，确保全部为 **运行中**。

---

## 步骤三：创建网站

### 3.1 添加网站

1. 点击左侧菜单 **网站**
2. 点击 **添加站点**
3. 填写信息：
   - **域名**: 留空或填写你的域名
   - **根目录**: `/www/wwwroot/com.book`
   - **数据库**: MySQL
   - 数据库名：`com_book`
   - 用户名：`com_book_user`
   - 密码：（点击随机生成，**请保存**）
   - **PHP 版本**: 选择 8.0 或 8.1
   - **数据库编码**: `utf8mb4`

4. 点击 **提交**

### 3.2 记录数据库信息

创建完成后会显示：
- 数据库名：`com_book`
- 用户名：`com_book_user`
- 密码：（随机生成的密码）
- 主机：`localhost`

**请复制保存，下一步需要修改配置！**

---

## 步骤四：上传项目文件

### 方法一：在线上传（推荐）

1. 点击左侧菜单 **文件**
2. 进入目录 `/www/wwwroot/com.book`
3. 点击 **上传** 按钮
4. 选择以下文件上传：
   ```
   .htaccess
   README.md
   TEST_RESULTS.md
   admin/           (整个文件夹)
   api/             (整个文件夹)
   config/          (整个文件夹)
   includes/        (整个文件夹)
   database.sql
   index.php
   index.html.bak
   ```

5. 上传完成后点击 **全部解压**（如果是压缩包）

### 方法二：Git 拉取

1. 在 **文件** 页面，进入 `/www/wwwroot/com.book`
2. 点击右上角 **终端**
3. 执行命令：
   ```bash
   cd /www/wwwroot/com.book
   git clone https://github.com/xsq0428/com.book.git temp
   mv temp/* .
   mv temp/.* .
   rm -rf temp
   chown -R www:www /www/wwwroot/com.book
   chmod -R 755 /www/wwwroot/com.book
   ```

### 方法三：BT 命令上传

1. 下载项目压缩包到本地
2. 通过 FTP 工具（如 FileZilla）上传到 `/www/wwwroot/com.book`
3. 解压并设置权限：
   ```bash
   cd /www/wwwroot/com.book
   unzip com.book.zip
   chown -R www:www /www/wwwroot/com.book
   chmod -R 755 /www/wwwroot/com.book
   ```

## 步骤五：配置数据库

### 5.1 导入数据库

1. 点击左侧菜单 **数据库**
2. 找到刚创建的数据库 `com_book`
3. 点击右侧 **管理** 进入 phpMyAdmin
4. 选择 `com_book` 数据库
5. 点击 **导入** 标签
6. 选择 `/www/wwwroot/com.book/database.sql` 文件
7. 点击 **执行**

### 5.2 修改数据库配置

1. 点击 **文件** 菜单
2. 进入 `/www/wwwroot/com.book/config/`
3. 找到 `database.php` 文件，点击 **编辑**
4. 修改以下内容（第三步保存的数据库信息）：

```php
<?php
define('DB_HOST', 'localhost');
define('DB_USER', 'com_book_user');      // 改为你的数据库用户名
define('DB_PASS', '你的数据库密码');      // 改为你的数据库密码
define('DB_NAME', 'com_book');           // 改为你的数据库名
define('DB_CHARSET', 'utf8mb4');
```

5. 点击 **保存**

## 步骤六：配置 PHP

### 6.1 PHP 设置

1. 点击左侧菜单 **软件商店**
2. 找到已安装的 PHP 版本（如 PHP 8.0）
3. 点击 **设置**
4. 进行以下配置：

#### 禁用函数
- 点击 **禁用函数** 标签
- 删除 `putenv`（项目需要）

#### 上传限制
- 点击 **php 上传限制**
- 修改 `upload_max_filesize` 为 `50M`
- 修改 `post_max_size` 为 `50M`

#### 超时时间
- 点击 **超时设置**
- 修改 `max_execution_time` 为 `300`

### 6.2 安装 PHP 扩展

在 PHP 管理页面，确保安装以下扩展：
- [x] fileinfo
- [x] mbstring
- [x] mysqli
- [x] pdo_mysql
- [x] openssl

## 步骤七：配置 Nginx/Apache

### Nginx 配置

1. 点击 **网站** 菜单
2. 找到你的网站，点击 **设置**
3. 点击 **伪静态** 标签
4. 选择 `ThinkPHP` 或自定义，粘贴以下内容：

```nginx
location / {
    if (!-e $request_filename){
        rewrite  ^(.*)$  /index.php?s=$1  last;   break;
    }
}

location ~ \.php$ {
    fastcgi_pass   unix:/tmp/php-cgi-80.sock;
    fastcgi_index  index.php;
    include fastcgi_params;
    fastcgi_split_path_info ^(.+\.php)(/.+)$;
    fastcgi_param  PATH_INFO $fastcgi_path_info;
    fastcgi_param  PATH_TRANSLATED $document_root$fastcgi_path_info;
    fastcgi_param  SCRIPT_FILENAME $document_root$fastcgi_script_name;
}

location ~* \.(js|css|png|jpg|jpeg|gif|gz|svg|mp4|ogg|ogv|webm|htc|xml|woff)$ {
    access_log off;
    expires max;
}

location ~ /\. {
    deny all;
}
```

5. 点击 **保存**

### Apache 配置

项目已包含 `.htaccess` 文件，无需额外配置。

## 步骤八：设置目录权限

### 8.1 执行命令

点击 **终端** 或 SSH 连接，执行：

```bash
cd /www/wwwroot/com.book

# 设置所有者
chown -R www:www /www/wwwroot/com.book

# 设置目录权限
find /www/wwwroot/com.book -type d -exec chmod 755 {} \;

# 设置文件权限
find /www/wwwroot/com.book -type f -exec chmod 644 {} \;

# 特殊目录权限
chmod 777 /www/wwwroot/com.book/config
```

### 8.2 检查权限

确保以下目录可写：
- [x] `/www/wwwroot/com.book/config/`
- [x] `/www/wwwroot/com.book/admin/`

## 常见问题

### Q1: 访问页面 500 错误

**原因**: PHP 版本不兼容或权限问题

**解决**:
```bash
# 检查 PHP 版本
php -v

# 确认使用 PHP 8.0+

# 检查权限
chown -R www:www /www/wwwroot/com.book
chmod -R 755 /www/wwwroot/com.book
```

### Q2: 数据库连接失败

**原因**: 数据库配置错误

**解决**:
1. 检查 `config/database.php` 文件
2. 确认数据库名、用户名、密码正确
3. 确认 MySQL 服务已启动

### Q3: 页面空白无内容

**原因**: 数据库表未导入

**解决**:
1. 登录 phpMyAdmin
2. 确认 `com_book` 数据库中有以下表：
   - `admins`
   - `urls`
   - `ads`
   - `announcements`
   - `settings`
   - `visit_logs`
3. 重新导入 `database.sql`

### Q4: 静态资源加载失败

**原因**: 路径配置错误

**解决**:
1. 检查网站根目录是否正确
2. 检查 Nginx/Apache 配置
3. 确保 `.htaccess` 文件存在（Apache）

### Q5: 后台登录 404

**原因**: 伪静态配置问题

**解决**:
- Nginx: 检查伪静态配置
- Apache: 确认 `.htaccess` 生效
